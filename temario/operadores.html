JavaScript - Operadores (Compraración, Logicos, Aritméticos)

Artiméticos

Los operadores aritméticos nos van a permitir realizar calculos matematicos

Suma

Suma: Para realizar una suma debemos utilizar el operador más (+)

12 + 3; // 15

Resta

Resta: Para realizar una resta debemos utilizar el operador menos (-)

12 - 3; // 9

Multiplicación

Multiplicación: Para realizar una multiplicación debemos utilizar el operador asterisco (*)

12 * 3; // 36

División

División: Para realizar una división debemos utilizar el operador barra (/)

12 / 3; // 4

Módulo

Módulo: Para obtener el módulo debemos utilizar el operador porcentaje (%)

Vale aclarar que el módulo es el resto de la división

12 % 3; // 0 11 % 3; // 2

Compraración

Los operadores de comparación van a devolver verdadero (true) o falso (false) dependiendo de como se evalue esa comparación

Mayor

Para hacer una comparación de mayor debemos utilizar el operador >, al utilizarlo diremos que el operando de la izquierda es mayor que el operando de la derecha, en caso de que esto sea cierto se devolvera true, caso contrario false. Veamos un ejemplo

18 > 12 // true

Estamos preguntando si el numero 18 es mayor que el numero 12, lo cual es cierto

Menor

Para hacer una comparación de menor debemos utilizar el operador <, al utilizarlo diremos que el operando de la izquierda es menor que el operando de la derecha, en caso de que esto sea cierto se devolvera true, caso contrario false. Veamos un ejemplo

18 < 12 // false

Estamos preguntando si el numero 18 es menor que el numero 12, lo cual es falso

Mayor o Igual / Menor o Igual

Puede darse el caso que en la comparación querramos tener en cuenta la igualdad, por ejemplo, si lo que queremos es hacer una verificación de mayoría de edad (teniendo en cuenta que esta empieza a la edad de los 18) tenemos las siguientes posibilidades

var edad = 18;

edad > 18; // false

edad > 17; // true

edad >= 18; // true

Si queremos saber si es menor, tenemos el caso contrario

var edad = 17;

edad < 17; // false

edad < 18; // true

edad <= 17; // true

Igualdad

Para hacer una comparación de igualdad no podemos utilizar solamente un igual (=) dado que este operador dentro de JavaScript es utilizado para la asignación es por ello que tenemos que recurrir a dos posibles operadores: doble igual (==) o triple igual (===), veamos un ejemplo y luego veamos la diferencia

12 == "12"; // true

12 === "12"; // false

Como podemos ver la primera igualdad (doble igual ==) no esta haciendo un chequeo por tipo de dato, lo que esta sucediendo es que en la comparación se esta haciendo una conversión de dato a uno en común y luego se comparan

Sin embargo en el la segunda igualdad (triple igual ===) es una igualdad estricta, no solo se fija en el valor sino que ademas el tipo de dato tiene que ser el mismo, como aca se esta comparando un number con un string su valor de igualdad sera falso

Desigualdad

Similar a lo que tuvimos en la igualdad habra una desigualdad y una desigualdad estricta donde el tipo de dato se tiene en cuenta

En esta caso la desigualdad es "no igual" (!=) y la desigualdad estricta será "no igual igual" (!==)

12 != "12"; // false

12 !== "12"; // true

En el primer caso, como se realiza una conversión de tipo de dato, los valores serán iguales por lo cual la desigualdad dara falsa, sin embargo, en el segundo caso, como los tipos de datos son distintos la desigualdad es verdadera sin analizar siquiera los valores

Logicos

Estos operadores nos permiten evaluar dos sentencias según sea su valor de verdad

Para poder comprender que resultado conseguiremos con cada uno de los operadores necesitamos tener bien en claro las tablas de la verdad, cada operador tiene asociada su propia tabla que se detalla en cada uno

Y (AND)

V F F

El operador AND (&&) será verdadero cuando sus dos proposiciones a evaluar son verdaderas

var nombre = 'jonathan';

var edad = 20;

(nombre === 'jonathan') && (edad >= 18); // true

(nombre === 'carlos') && (edad >= 18); // false
(nombre === 'jonathan') AND (&&) (edad >= 18)
V V V

En la primer evaluación de AND tanto (nombre === 'jonathan') como (edad >= 18) es verdadero, por lo cual el resultado del AND será verdadero

(nombre === 'carlos') AND (&&) (edad >= 18)
F F V

Sin embargo en la segunda evaluación de AND tanto (nombre === 'carlos') es falso, por lo cual el resultado de la evaluación del AND será tambien falso

A diferencia del operador AND el operador OR (||) sera falso cuando sus dos proposiciones sean falsas. Por lo cual decimos que es suficiente que al menos un elemento sea verdadero para que el OR de verdadero

var nombre = 'jonathan';

var edad = 20;

(nombre === 'jonathan') || (edad >= 18); // true

(nombre === 'carlos') || (edad >= 18); // true

(nombre === 'carlos') || (edad >= 28); // false
(nombre === 'jonathan') OR (||) (edad >= 18)
V V V
(nombre === 'carlos') OR (||) (edad >= 18)
