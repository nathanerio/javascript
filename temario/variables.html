JavaScript - Variables (Var, Let, Const) y Tipos de Datos

Declaración

Para poder declarar una variable vamos a utilizar la palabra reservada var de la siguiente forma

var nombre;

Acabamos de declarar una variable llamada nombre

En el caso de querer asignarle un valor a esa variable podemos inicializarla de la siguiente forma:

nombre = 'jonathan';

El simbolo de igual se utilizar para la asignación, es decir, que la variable que esta a la izquierda del igual (nombre) se le asignará como valor el texto que esta a la derecha ('jonathan')

Se puede crear una variable e inicializarla en la misma linea de la siguiente forma:

var nombre = 'jonathan';

De esta forma, no solo creamos una variable sino que además ya le asignamos un valor

Tipo de Datos

Dentro de JavaScript tenemos principalmente dos tipos de datos los primitivos y los objetos

Tipo de Datos: Primitivos

Dentro de esta categoría vamos a encontrar los String, Boolean, Number, Null, Undefined

Undefined

undefined: Es el valor que tiene por defecto una variable que no fue inicializada

var nombre; var nombre = undefined;

Null

null: Es un tipo de dato que representa un valor nulo, la diferencia con undefined es que este valor debe ser asignado y dentro de nuestro código representa la nulificación del valor de la variable, si hablamos de punteros, diriamos que no apunta a nada

var nombre = null;

String

string: Para poder establecer un valor de string o cadena, dentro de JavaScript tenemos dos posibilidades, o escribimos la cadena dentro de comillas simples ('') o dentro de comillas dobles (""), popularmente la mas utilizadas son las comillas simples

var nombre = 'jonathan'; var nombre = "jonathan";

Number

number: Podemos asignar a una variable un valor entero (12) o un valor de coma flotante como ser (12.3), el delimitador de los decimales dentro de JavaScript será entonces el punto (.)

var edad = 20; var pi = 3.14;

Dentro de los posibles valores numericos tenemos 3 adicionales a los numeros y estos son: Infinity, -Infinity y NaN

El valor Infinity o su negativo puede obtenerse al realizar una división entre un numero cualquiera y el 0, es decir, si 12 lo dividimos por 0 nos dará infinito, en el caso de que el numero sea negativo, nos dará menos infinito

12 / 0; // Infinity -12 / 0; // -Infinity

El valor NaN representa un Not A Number, es decir, un 'numero' que no es un numero, esto puede obtenerse de varias formas, una de ellas es la division de 0 entre 0

0 / 0; // NaN

Boolean

boolean: Este tipo de dato solo puede tomar uno de dos valores posibles, estos son verdadero (true) o falso (false)

var verdadero = true; var falso = false;

Tipo de Datos: Objetos

Array

Podemos definir a un array como un tipo de dato en el cual almacenaremos varios valores (pueden ser primitivos u objetos), del mismo o diferente tipo que serán accedidos mediante un indice numérico

Los arrays se definen utilizando los corchetes ("[ ]") y dentro de ellos los valores se separan por comas (",") var vector = [ 1, 2, 3, 4, 5 ]; var nombres = [ 'Juan', 'Pedro', 'Pablo' ]; var varios = [ 'Hola', 12, true ];

Para poder obtener un valor dentro del array debemos acceder a la posición del elemento en cuestión, para ello usaremos el nombre del array seguido de unos corchetes y dentro de ellos el numero de indice del elemento, por ejmplo, si quisiese obtener el nombre "Pedro" del array "nombres" definido arriba debería utilizar como indice el uno (1), esto se debe a que los arrays se numeran desde el cero (0) en adelante, es decir, 0, 1, 2, 3, 4, etc.

nombres[1]; // "Pedro"

Object

Los objetos en JavaScript se utilizar para almacenar datos de tipo clave / valor, es decir, que necesitamos definir una clave (con la cual se accederá al dato) y un valor para la misma, los tipos de datos de los valores pueden ser primitivos u objetos y pueden ser todos del mismo tipo o distintos

Cada par de clave / valor (key / value) debe ser separado por una coma (",")

Generalmente estas claves suelen ser llamadas "propiedades" o "metodos", en el caso de que el valor de la clave sea una función diremos que es un "metodo" del objeto, para los otros casos decimos "propiedad" del objeto

var persona = { nombre: 'Jonathan', edad: 20 };

Para acceder al valor de una propiedad en particular, simplement utilizamos el operador punto ("."), esto significa que deberemos escribir primero el nombre de la variable (que es el objeto propiamente dicho) luego un punto para luego escribir el valor de la clave (key) a la que queremos acceder

En caso de querer obtener la edad de la persona que definimos anteriormente deberíamos escribir lo siguiente:

persona.edad; // 20

Como dijimos anteriormente, un posible valor puede ser un objeto por lo cual podriamos tener lo siguiente:

var curso = { nombre: 'JavaScript', clases: [ { nombre: 'clase 1', temario: 'variables' } ] };

En esta caso estamos definiendo un objeto curso, que tiene como propiedad el nombre del mismo ('JavaScript'), y ademas tiene otra propiedad que es un Array, este ultimo tiene un solo elemento que es un objeto conformado por dos propiedades, el nombre ("Clase 1") y el temario ("variables")

En caso de querer acceder al temario de la clase 1 del curso haremos lo siguiente:

curso.clases[0].temario; // "variables"

Variable (curso), propiedad (clases), primer elemento del array ([0]), propiedad (temario)

Function

Las funciones serán analizadas en el aparatado de Funciones (Function), sin embargo vamos a ver como poder asignarlas a una variable

var fn = function () { /* codigo... */ }

Luego podemos invocar a esa función de la siguiente manera

fn();

Let y Const

En ECMAScript6 se agregaron dos nuevas palabras reservadas para definir variables, estas son let y const

Let

Permite definir una variable, lo que significa que su valor puede cambiar durante la ejecución del programa

let numero = 12;
numero = 24;

Const

Permite definir una constante, lo que significa que su valor no puede cambiar durante la ejecución del programa

const pi = 3.14;

En caso de que tratemos de cambiar el valor de una constante nos va a dar un error

pi = 3.1415; TypeError: Assignment to constant variable.

Otro error lo tendriamos si tratamos de definir una constante sin asignarle valor en su definción

const impuesto; SyntaxError: Missing initializer in const declaration

Como puede verse, siempre que se defina una constante hay que asignarle el valor que llevará dentro de la ejecución del programa

Alcance de las Variables

Dependiendo de en donde creamos una variable define donde va a ser utilizable la misma, eso significa que puede que una variable no este disponible en cualquier parte de nuestro codigo y a eso es lo que se denomena alcance (scope)

En un principio diremos que las variables definidas con var seran variables con alcance (scope) de función, sin embargo, si estas no se definen dentro de ninguna función entonces seran variables globales

En cambio decimos que las variables definidas con let y const seran variables con alcance (scope) de bloque, si estas no se definen dentro de ningun bloque puntual diremos que son variables locales, no decimos que son globales porque no se asocian al objeto global window

var nombre = 'jonathan';

function mostrarNombre() { console.log(nombre); }

mostrarNombre(); // "jonathan";

Como se puede observar en el código anterior, a pesar de haber definido la variable por fuera de la función, puede ser utilizada si ningun problema dentro de la misma, esto se debe a que la variable nombre es global

Ahora, si nosotros definimos una variable dentro de la función y tratamos de utilizarla por fuera de la misma

function mostrarNombre() { var local = 'variable local'; console.log(local); }

mostrarNombre(); // "variable local";
console.log(local); ReferenceError: local is not defined

Nos terminará dado un error de referencia, dado que la variable local no existe en ese punto, puesto que la variable local, como su nombre lo indica, es local a la función mostrarNombre

A diferencia de var las variables definidas por let y const diremos que tiene un alcance mas limitado, dado que su limite esta dado por un bloque de codigo, crearemos a continuación un bloque de codigo anonimo donde definiremos una variable let dentro y trataremos de utilizarla por fuera de la misma

{ let apellido = 'j'; }
console.log(apellido); ReferenceError: apellido is not defined

Sin embargo el mismo código con var funcionaría

{ var apellido = 'j'; }
console.log(apellido); // "j"

Estos "bloques" lo encontraremos en JavaScript en varios lugares como ser los condicionales (If / Else), los Switch, los ciclos de repetición (While, Do While, For), las funciones (Function), etc.
